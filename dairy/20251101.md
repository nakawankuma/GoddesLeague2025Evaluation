# 2025/11/01 作業記録

## Issue #7: HTML検証ツールの導入

### 作業内容
- HTML検証ツール（html-validate）を導入
- コミット前に自動的にHTMLを検証するpre-commitフックを設定
- Windows/Linux両環境で動作するようにhuskyを使用

### 導入したツール
1. **html-validate** - HTML構文チェックツール
2. **husky** - Gitフック管理ツール

### 設定ファイル
1. **package.json**
   - `lint`: HTML検証スクリプトを追加
   - `lint:fix`: 自動修正スクリプトを追加
   - `prepare`: husky初期化スクリプトを追加
   - devDependenciesに`html-validate`と`husky`を追加

2. **.htmlvalidate.json**
   - HTML検証ルールの設定
   - 動的コンテンツ用にいくつかのルールを緩和
     - `no-implicit-button-type`: OFF（動的に生成されるボタン）
     - `text-content`: OFF（JavaScriptで内容が設定される要素）
     - `empty-heading`: OFF（動的に設定される見出し）
     - `element-permitted-content`: OFF（スタイル要素の配置）

3. **.husky/pre-commit**
   - コミット前に`npm run lint`を自動実行

### 使い方
```bash
# 手動でHTML検証を実行
npm run lint

# 自動修正を試みる
npm run lint:fix

# コミット時に自動実行される
git commit -m "message"
```

### クロスプラットフォーム対応
- huskyを使用することでWindows（PowerShell）とLinux（Bash）両方で動作
- 従来のシェルスクリプトベースのGitフックはWindows環境で問題が発生するため、huskyで統一
